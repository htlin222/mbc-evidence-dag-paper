## 2. Round 3 asks (bar-raising)

1. **ACTION: change the Methods sentence "treating each definition as a set of operational tokens parsed from the eligibility text" at `manuscript/main.tex:107` to "treating each definition as a set of operational tokens hand-coded from the canonical primary publication of each trial against the fixed token vocabulary in Supplementary Table S4 (released as `analysis/06_compute_odi.py`)", because `06_compute_odi.py:22-57` is a static `DEFINITIONS` dict with no parser, and this is now the **third reviewer-cycle ask on the same sentence** (R1-#16 → R2-#11 → R3-#1).** This sentence is detectable from a first-page Methods read and is the single remaining overclaim of pipeline automation in the manuscript. One-line fix.

2. **ACTION: justify the per-node `year` values in `analysis/03_encode_guideline.py` against the published guideline-version history, because G3 (post-endo CDK4/6i+fulvestrant) is encoded as `year: 2016` and the temporal-precedence test at `05_compute_efdpr.py:146-147` therefore *excludes* MONARCH-2 (year_pc 2017) and MONALEESA-3 (year_pc 2018) from G3's support set, even though ESMO 2024 explicitly cites all three trials at G3.** The result is that `efdpr.json:28-30` reports G3 support = `[NCT01942135]` (PALOMA-3 only), not the three trials in `03_encode_guideline.py:57`. This is internal to the repository and has no effect on the headline EFDPR (G3 is still evidence-supported), but it means the **temporal-precedence test is silently reading "year of node introduction" as "year of earliest supporting evidence", which is a different semantic and one the reviewer will catch on a diff between `cited_trials` and `per_node_support`**. Either (a) document this semantic in Methods, naming the field `node_introduction_year`, or (b) use a single guideline-freeze year (2024) for the temporal-precedence test and rerun. R3-#2 is the Round-1 #15 / #30 ask still alive.

3. **ACTION: lift the corpus-selection-bias disclosure from `supplement.tex:159-163` into Methods §2.1 of the main text (one sentence is sufficient: "The 16-trial corpus is seeded from ESMO and ASCO HR+/HER2- mBC guideline reference lists, which biases inclusion toward trials the guidelines already cite; EFDPR is therefore reported as a guideline-conditional estimate."), because the present `main.tex:118` ("drawn from the ESMO and ASCO mBC guideline reference lists") states the *fact* but not the *consequence*, and a JCO CCI reviewer reading the main text without the supplement cannot judge external validity of the EFDPR estimate.** This is R2-#5, partially addressed but not closed.

4. **ACTION: render Supplementary Tables S2, S3, S4 as formatted LaTeX longtables in `supplement.tex` (rather than the current "released as JSON" prose), because Round 1 ask #24, Round 2 ask #14, and Round 3 still see the same substitution at `supplement.tex:70-96`.** A JSON file is not a supplementary table. The mechanical fix is to add three `\begin{longtable}` blocks generated from the existing JSON. For S2: NCT, trial name, prior-state tokens, biomarker, drug class, year, guideline-target node. For S3: G-id, state, biomarker, recommended classes, grade, cited NCTs, year. For S4: biomarker variable, trial, operational tokens. The per-node `Supp(g)` set (already computed in `efdpr.json` as `per_node_support`) should also be a longtable in this supplement.

5. **ACTION: add a one-sentence Methods note disclosing the modal-stratum encoding rule for trials with mixed-line enrolment, because `02_extract_eligibility.py:128-132` encodes MONALEESA-3 (which enrolled both 1L and post-endo populations) as `post_endo: True`, routing the edge to G3 and excluding it from G1 support; the inline code comment names the choice but Methods is silent.** A neutral sentence in Methods §2.2 is enough: "For trials with mixed line-of-therapy enrolment (e.g. MONALEESA-3), the modal stratum is used as the canonical source state; subgroup-readout support for the non-modal stratum is captured in the `subgroup_readouts` field and is accessible only under liberal tolerance." Without this, a reader who diffs `cited_trials` against `per_node_support` will see G1 supported by 3 trials (MONALEESA-2, PALOMA-2, MONARCH-3) and G3 supported by 1 trial (PALOMA-3 only, per R3-#2), and will reasonably ask where MONALEESA-3 went.

6. **ACTION: remove OpenAlex from both the preregistration data-source list (`docs/prereg.md:49`) and the Acknowledgements (`manuscript/main.tex:202`), or supply the OpenAlex query and result file, because there is no OpenAlex API call anywhere in `analysis/0[1-8]_*.py` and no `openalex_*.json` under `data/raw/`.** The supplement deviation list at `supplement.tex:103-164` lists nine deviations but does *not* include the OpenAlex non-use. This is Round 1 ask #27 → Round 2 ask #15(d), still alive. Three-line fix: either delete the OpenAlex line from the prereg, or add a supplement-deviations bullet "OpenAlex was preregistered but not invoked in the present pilot; deferred to the production extension".

7. **ACTION: replace the bootstrap-percentile indexing `lo_idx = max(0, int(0.025*n)-1); hi_idx = min(n-1, int(0.975*n))` at `analysis/05_compute_efdpr.py:184-187` (and the identical block at `06_compute_odi.py:79-81`) with `numpy.quantile(estimates, [0.025, 0.975], method='linear')`, because the current expression is not the textbook percentile estimator it claims to be (the comment at `05_compute_efdpr.py:184-185` cites "ceil(alpha*n)-1" for lower but the code is `int(0.025*n)-1 = 24`, which is `floor(25)-1`, i.e. *off by one in the opposite direction*), and at $n=1000$ with many bootstrap replicates collapsing to the same $k/16$ value (the present regime) the discrepancy can move the reported lower-CI by one discrete step ($\pm 0.0625$ at $n=16$).** This is Round 1 ask #7 → Round 2 ask #9, still alive. The five-line fix is mechanical.

8. **ACTION: add a Methods or Limitations sentence disclosing the singleton-token Jaccard mechanical-inflation issue for the prior-CDK4/6i ODI, because `06_compute_odi.py:48-56` shows that six of the seven trials have *singleton* operational token sets (EMERALD `{"prior_CDK4_6i_required"}`, CAPItello-291 `{"prior_CDK4_6i_allowed"}`, PALOMA-3 `{"prior_endo_progression"}`, MONARCH-2 `{"prior_endo_progression"}`, postMONARCH `{"prior_CDK4_6i_required"}`, INAVO120 `{"endo_resistance_window_12mo"}`), so the Jaccard distance is 1 whenever the singletons differ; this is what drives 18/21 pairs in `odi.json:100-205` to Jaccard distance = 1.0 and gives the headline ODI = 0.91.** This is Round 1 ask #22 → Round 2 ask #13, still alive. The Limitations paragraph in `discussion.tex:26` mentions corpus and hand-curation limitations but not the ODI's known measurement artefact. One-sentence addition is sufficient: "The headline prior-CDK4/6i ODI is dominated by binary disagreement on a single canonical token per trial and overstates the *graded* operational discordance an ordinal score would measure."

9. **ACTION: fix the abstract phrasing "16 pivotal phase 2/3 HR+/HER2- mBC trials (plus one PARP-inhibitor reference trial)" at `manuscript/main.tex:55` so it does not read as 16+1 = 17 trials, because the actual corpus is 16 trials *including* OlympiAD; the abstract phrasing implies a 17th.** The Results §3.1 sentence at `main.tex:118` ("Fifteen trials enrolled HR+/HER2- ... and one (OlympiAD) was the PARP-inhibitor reference trial for the gBRCAm guideline node") is the correct framing and should be backported to the abstract. Also align Table 1 `tab_cohort.tex` row "ESMO 2024 decision nodes (HR+/HER2- subset) & 16": G16 (`gBRCAmut/HER2-`) is not in the HR+/HER2- subset, so the label is mildly inconsistent with the supplement-disclosed scope expansion. Either re-label the table row "ESMO 2024 decision nodes (HR+/HER2- + gBRCAm/HER2- subset) & 16" or split the count "HR+/HER2- nodes & 15; gBRCAm/HER2- nodes & 1".

10. **ACTION: delete the `excluded` field from `analysis/02_extract_eligibility.py` (and `data/processed/trials_structured.json`) or wire it into `04_build_dag.py` / `05_compute_efdpr.py` and report its effect on EFDPR, because the field is currently populated for ten of the sixteen trials (e.g. PALOMA-3: `excluded: {"post_cdk46i": True}` at line 36; MONARCH-2: `excluded: {"post_cdk46i": True, "post_chemo_metastatic_any": True}` at line 45; MONALEESA-2: `excluded: {"post_endo_metastatic_any": True, "post_chemo_metastatic_any": True}` at line 94) and never read; in the present concordance algorithm a trial that *excludes* a prior state is therefore equivalent to a trial that is *silent* on it.** This is Round 1 ask #6 → Round 2 ask #8, still alive. Either deletion or wiring is acceptable; the present state (populated but ignored) is a correctness defect that is detectable from any diff of the JSON files.

11. **ACTION: add to Methods §2.7 (Reproducibility and software) the pinned package versions and a one-line driver-command invocation (e.g. "`uv run python analysis/01_fetch_trials.py && ... && python analysis/08_sensitivity_no_g16.py`" or a `make all` reference to `analysis/Makefile`), because `main.tex:112-113` lists "networkx 3.x, pandas 2.x, numpy 1.26+" without exact pins and gives no driver command.** This is Round 1 ask #26 → Round 2 ask #18, still alive. A JCO CCI reviewer reading the Code Availability statement expects either a `pyproject.toml`+`uv.lock` reference or a fully-pinned `requirements.txt` in the repository, and a one-line command to reproduce all figures from the frozen JSON.

12. **ACTION: render Supplementary Table S1 — the per-node `Supp(g)` table — as a formatted longtable in `supplement.tex`, alongside the existing TRIPOD-style reporting checklist, because `efdpr.json:19-60` (and the analogous blocks for ESCAT and liberal) contains the per-node supporting NCT list for every guideline node, and the Results paragraph at `main.tex:135` enumerates 5 evidence-free node IDs without ever showing the supporting-edges for the 11 evidence-supported nodes; the reader cannot verify a single concordance call from the manuscript alone.** This is Round 1 ask #25, repeatedly deferred. A 16-row table (G-id, state, biomarker, supporting NCTs under strict, supporting NCTs under liberal) is the single most useful supplementary table this manuscript could produce.

- **15-node sensitivity (R2-#2 / Q5).** `analysis/08_sensitivity_no_g16.py` correctly drops G16 (`gls_15 = [g for g in gls if g["node_id"] != "G16"]`), and the resulting `efdpr_sensitivity_no_g16.json` shows strict: $k=5$, $n=15$, point=0.333, CP CI [0.118, 0.616], $P=0.314$; liberal: $k=4$, $n=15$, point=0.267, CP CI [0.078, 0.551], $P=0.539$. The manuscript at `main.tex:172` rounds these to 0.33 / 0.27 / $P=0.31$ / $P=0.54$, which match the JSON. Numerator (5 strict, 4 liberal) and denominator (15) are correctly described. No subtle inconsistency.

- **Drug-class equivalence table (R2-#3/#4).** `05_compute_efdpr.py:118-135` is identity-only except for `chemotherapy ⇄ chemotherapy single agent` and `PARPi (olaparib) ⇄ PARPi (talazoparib)`. The Round-2 prefix-shortcut concern is fully resolved.

The Methods sentence at `main.tex:100` is faithful to the code. - **Pre-registered exact-binomial test.** `05_compute_efdpr.py:191-200` is a textbook one-sided exact binomial implementation; the Table-2 P-values match a hand-calculation; the non-rejection headline is correct under all three tolerances.

- **Five S1–S4 reporting checklists.** Supplementary Table S1 (TRIPOD-style + graph-encoding addendum) is materialised as a longtable at `supplement.tex:18-68`. Items 1–29 and G1–G6 are populated. S2/S3/S4 are *not* materialised as formatted tables (see R3-#4 and R3-#12).
