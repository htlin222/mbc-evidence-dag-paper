# Round 1 Methods Review

**Reviewer role:** adversarial methods reviewer
**Target journal:** JCO Clinical Cancer Informatics (Original Report)
**Manuscript:** `manuscript/main.tex` + `manuscript/discussion.tex`
**Materials reviewed:** `docs/prereg.md`, `analysis/01_*` through `analysis/07_*`, `data/processed/*.json`, `data/results/*.json`, `manuscript/JOURNAL.md`.

The pilot is honestly framed and the release artefacts are unusually complete for a 14-trial pilot. The methodological problems below are nevertheless severe enough that several manuscript claims are not currently supported by the released code, and at least two are contradicted by it. The asks are ordered by severity.

---

## 1. Code–paper contradictions (must fix; blocking)

1. **ACTION: change the Methods sentence "The eligibility text returned by ClinicalTrials.gov was parsed into this schema using a large-language-model (LLM) extraction pipeline with chain-of-thought prompts, two in-context exemplars, and per-field confidence labels. A hold-out validation set of trials with clinical-investigator-annotated ground truth was used to estimate per-field accuracy and Cohen's $\kappa$." (`manuscript/main.tex:91`) to an accurate description of what the released code actually does, because `analysis/02_extract_eligibility.py:2-12` explicitly states "eligibility coding is performed by hand-curated assignment" and there is no LLM call, no exemplars, no validation set, and no Cohen's κ anywhere in the repository.** This is the single most damaging discrepancy in the manuscript; the current text misrepresents the artefact under review.

2. **ACTION: change the Methods description of the state-match criterion on `manuscript/main.tex:100` from "the source state $u$ equals the guideline-node patient sub-state" to "the trial-required prior states are a superset of the guideline-node prior states (a trial requiring post-CDK4/6i + post-endo can support a guideline node requiring only post-CDK4/6i)", because `analysis/05_compute_efdpr.py:51-53` implements `_state_tokens(gl_state).issubset(_state_tokens(edge_state))`, not set equality.** The current prose claims equality; the code implements containment. These are not the same rule and they give different EFDPR values.

3. **ACTION: justify the superset rule explicitly in Methods and run a `state-equality` sensitivity analysis alongside strict/ESCAT/liberal, because the superset rule systematically reduces EFDPR by accepting more-restrictive trials as concordant for less-restrictive guideline nodes; without this, the reader cannot tell whether the 0.467 estimate is an artefact of the relaxation.** The relaxation is defensible but it is currently silent.

4. **ACTION: fix the CAPItello-291 record in `analysis/02_extract_eligibility.py:71-78` so that `prior_state.post_cdk46i` is `None` (allowed-but-not-required), not `False` (excluded), because the manuscript itself (`discussion.tex:13`) and the ODI token in `analysis/06_compute_odi.py:50` correctly describe CAPItello-291 as permitting prior CDK4/6i, and because the current `False` value is what causes the trial to fail state-match against G6 in the strict run.** The numeric finding for G6 may still hold even after the fix, but the current encoding is wrong on the face of it and the discrepancy is internal to the repository.

5. **ACTION: fix the MONALEESA-3 prior-state encoding in `analysis/02_extract_eligibility.py:98-106`, because `prior_state.post_endo` is set to `None` and `analysis/04_build_dag.py:24` only treats `is True` as post-endo, so MONALEESA-3 is routed to the first-line node (see `data/processed/dag_edges.json:93-101`) even though its `guideline_target_node` is `post-endo|HR+/HER2-`.** This affects G3 incident-edge count and the in-degree statistics in Results §3.1.

6. **ACTION: either delete the `excluded` field from `analysis/02_extract_eligibility.py` records or use it in `analysis/04_build_dag.py` / `analysis/05_compute_efdpr.py`, because at present the field is populated but never read, so a trial that excludes a prior state cannot be distinguished from one that is silent on it.** This is a correctness defect, not cosmetic: MONALEESA-2's "excluded: post_endo_metastatic_any" is silently ignored.

## 2. EFDPR statistic and bootstrap CI

7. **ACTION: replace the percentile-bootstrap implementation `lo = estimates[int(0.025 * n)]; hi = estimates[int(0.975 * n) - 1]` in `analysis/05_compute_efdpr.py:167-168` with `numpy.quantile(estimates, [0.025, 0.975], method='linear')` or a documented BCa interval, because the current expression returns the 26th-smallest and 975th-smallest values (asymmetric), is not a standard percentile estimator, and the discrepancy is non-trivial when many bootstrap replicates collapse to the same k/15 value.** State the chosen method in Methods.

8. **ACTION: change Methods (`manuscript/main.tex:104`) to state that the bootstrap resamples **guideline decision nodes** (not trials and not edges) with replacement, and add a one-sentence justification that this estimates the sampling variability of $|\mathcal{G}|$ rather than of the trial corpus, because the current text is ambiguous and a reader could reasonably believe the bootstrap propagates uncertainty in the trial structuring.** It does not; trial-side error is not propagated at all.

9. **ACTION: add a sentence to Methods that explicitly disclaims independence of the 15 nodes (G6, G7, G9, G15 all sit at post-CDK4/6i biomarker-stratified positions and share guideline-citation structure) and report Wilson or Clopper–Pearson exact-binomial CIs alongside the bootstrap CI, because the bootstrap-on-nodes procedure assumes exchangeability and the resulting interval is narrower than it should be when nodes cluster.** With $n=15$ a Clopper–Pearson interval for 7/15 is 0.213–0.736; if that interval also excludes 0.25 the conclusion is more robust than the bootstrap alone implies.

10. **ACTION: change the pre-registered hypothesis test description on `manuscript/main.tex:55-56` from "the strict 95\% CI excluded the pre-registered null of 0.25" to a single, named test — either a one-sided exact-binomial test against $p_0=0.25$ or "the lower bound of the 95\% Clopper–Pearson interval exceeded 0.25" — and run it, because `docs/prereg.md:13` actually pre-specifies "one-sided exact binomial $p < 0.05$" as the rejection rule and that test was never executed in `analysis/05_compute_efdpr.py`.** The preregistered test and the test that was run are not the same.

11. **ACTION: report the binomial-test $p$-value for $H_0:\text{EFDPR}\le 0.25$ in Table \ref{tab:efdpr} (or in the EFDPR results paragraph), because the preregistration commits to a $p$-value-based decision rule and the manuscript currently substitutes a CI-overlap rule.** With 7/15 and $p_0=0.25$, $P(X\ge 7) \approx 0.066$ — note that this is above 0.05, which materially weakens the "threshold rejected" claim in `manuscript/tab_efdpr.tex` and changes the headline finding from "rejection" to "marginal".

12. **ACTION: add a brief Methods paragraph noting the discrete-resolution constraint that with $|\mathcal{G}|=15$ the EFDPR can only take values $k/15$ at granularity 0.0667 and the lower CI bound has at most ~5 distinct attainable values in (0.20, 0.40), because the threshold rejection at lower CI = 0.267 is only one node-reclassification away from the threshold at lower CI = 0.200.** This belongs in the limitations of the EFDPR statistic, not buried in the Discussion's "corpus size sensitivity" paragraph.

## 3. Concordance algorithm — operationalisation

13. **ACTION: change the drug-class family-equivalence block in `analysis/05_compute_efdpr.py:117-126` so that the prefix match for `"CDK4/6i"` does not silently accept `"CDK4/6i + fulvestrant (post-CDK4/6i)"` (postMONARCH) as concordant with a first-line `CDK4/6i + AI` recommendation, because the current `startswith("CDK4/6i")` rule collapses partner-drug and treatment-line into a single class token and erodes the precision of the drug-match criterion.** Use the full drug class string with an explicit equivalence table, and document the equivalences in Methods or in a supplementary table.

14. **ACTION: add a one-line definition of "registrational biomarker-subgroup readout" to Methods alongside the liberal-tolerance description on `manuscript/main.tex:56`, and tie it to an explicit field (`subgroup_readouts` in `data/processed/dag_edges.json`), because the term currently appears only in the prose and a reader cannot tell from the manuscript which trials qualify or how the classification was made.** Right now this is investigator-coded with no audit trail.

15. **ACTION: add to Methods the explicit definition of guideline-version year used for the temporal-precedence check, because `analysis/05_compute_efdpr.py:130-131` compares `edge_year <= gl_year` where `gl_year` is the analyst-assigned `year` in `analysis/03_encode_guideline.py` (range 2016–2024) but the manuscript Methods describes "the guideline-version year" as a single value, which is ambiguous when the ESMO 2024 update incorporates 2018 first-line recommendations.** Either fix the field name to `node_introduction_year` or define what `year` means per node.

16. **ACTION: replace the claim in Methods that biomarker tokens are "parsed from the eligibility text" (`manuscript/main.tex:107`) with "hand-coded from the canonical primary publication of each trial against a fixed token vocabulary (Supplementary Table S4)", because `analysis/06_compute_odi.py:29-57` shows that the per-trial token sets are statically declared, not parsed, and there is no parser in the repository.** As written, the Methods overclaims pipeline automation.

## 4. Preregistration honored vs. silently violated

17. **ACTION: add an explicit "Deviations from preregistration" subsection to Methods (or to the existing Statistical analysis subsection at `manuscript/main.tex:109`) that lists, at minimum: (a) corpus expansion from 6 trials / 7-node pilot to 14 trials / 15 nodes after preregistration, (b) substitution of CI-overlap rule for the prereg's exact-binomial test, (c) omission of pre-registered secondary outcome S2 (temporal evidence lag), (d) omission of pre-registered secondary outcome S3 (subgroup coverage), (e) omission of multiple-testing BH correction across S1/S2/S3, (f) omission of pre-registered NCCN sensitivity analysis (E1), because `docs/prereg.md:57` commits to "Any deviation from this preregistration will be reported in the manuscript with a paragraph explaining the deviation and justification" and none of these deviations is currently disclosed.**

18. **ACTION: justify the corpus-selection rule "drawn from ESMO 2024 reference list and ASCO 2022 guideline update reference list" (`analysis/01_fetch_trials.py:24-26`) in Methods, because selecting trials from the guideline reference lists is circular for an analysis whose outcome is guideline–trial concordance: trials not cited by guidelines are by construction excluded from the corpus, biasing EFDPR upward toward composition-only nodes and downward toward cited-and-matched nodes.** Either rerun on a search-based corpus (e.g., ClinicalTrials.gov query with the same eligibility filters and no guideline-list filter) or label this as a guideline-conditional EFDPR throughout.

19. **ACTION: add a paragraph to Methods describing the inclusion criterion that distinguishes "pivotal" from non-pivotal phase 2/3 trials, because the prereg refers to "phase 2/3 HR+/HER2- mBC trials, 2015–2026" (a search-defined corpus) but the realised corpus is 14 hand-picked trials (`analysis/01_fetch_trials.py:27-44`), and the manuscript uses "pivotal" without defining it (regulatory-registrational? FDA-label-supporting? guideline-cited?).** Without this, a reader cannot replicate the corpus.

## 5. Pilot scope — claims the 14/15 design cannot support

20. **ACTION: change the abstract sentence "EFDPR was 0.400 (95\% CI 0.200--0.667) and the lower CI bound touched the pre-registered threshold" (`manuscript/main.tex:56`) and the parallel Results sentence (`manuscript/main.tex:135`) to acknowledge that under liberal tolerance the null is not rejected (lower bound = 0.200 < 0.25), because "touched" is a euphemism that misrepresents a non-rejection.** This is consistent with what `discussion.tex:10` already says ("not robust to all tolerance choices"); the abstract should match.

21. **ACTION: move the claim "Evidence-free nodes clustered at post-CDK4/6i biomarker-stratified positions" from the abstract (`manuscript/main.tex:56`) to the Results as a descriptive observation, and remove the implicit causal framing, because with 7 evidence-free nodes in a 15-node tree the clustering is an artefact of where biomarker-stratification appears in the guideline tree (post-CDK4/6i is where most biomarker substratification happens), not a statistical test of cluster non-uniformity.** Alternatively, run a permutation test for spatial clustering of evidence-free nodes against the guideline tree's biomarker-stratification distribution.

22. **ACTION: remove or qualify the abstract claim that ODI = 0.91 reflects "the absence of a shared operational standard" (`manuscript/main.tex:152`), because ODI was computed across only 7 trials with token sets of size 1 (e.g., PALOMA-3 = {"prior_endo_progression"}, EMERALD = {"prior_CDK4_6i_required"}); when token sets are singletons, the Jaccard distance is 1 whenever the singletons differ, which mechanically inflates ODI without measuring the breadth of disagreement.** Either expand the token vocabulary so trials share at least some tokens or use an information-theoretic divergence that is well-defined for singletons.

23. **ACTION: add to Methods the rationale for treating each trial's prior-CDK4/6i operationalisation as a single token set rather than a structured tuple (required / allowed / excluded / window), because the current ODI for prior-CDK4/6i compares set membership of strings like "prior_CDK4_6i_required" vs. "prior_CDK4_6i_allowed" as if they were unrelated, which loses the natural ordering on the allow/require axis and inflates Jaccard distance.** A simple categorical-ordinal score (with explicit semantics) would be more informative.

## 7. Reproducibility

26. **ACTION: add to the `Reproducibility and software` subsection (`manuscript/main.tex:112-113`) a pinned dependency manifest reference (`pyproject.toml` and `uv.lock`) and a single-command invocation that runs `01_fetch_trials → … → 07_figures`, because currently a reader sees library names but no exact versions and no driver script; the `analysis/Makefile` is mentioned nowhere in the manuscript even though it appears to exist (referenced in CLAUDE delegation rule but not in the analysis dir; verify).** Also pin the random seed value (currently 20260516 in `analysis/05_compute_efdpr.py:159`) in Methods.

27. **ACTION: change "OpenAlex — query freeze date 2026-05-16" in `docs/prereg.md:49` and any manuscript reference to OpenAlex (Acknowledgements at `manuscript/main.tex:199`) to either supply the OpenAlex query and result file or remove the OpenAlex mention, because no OpenAlex call exists in `analysis/0[1-7]_*.py` and no `openalex_*.json` exists under `data/raw/`.** Promising a data source you didn't use is a reproducibility-claim defect.

28. **ACTION: document in Methods that target-node construction (`analysis/04_build_dag.py:55-58`) appends the drug-class string to source-state ("post-endo|after:CDK4/6i + fulvestrant") rather than computing a canonical post-treatment state token, with the consequence that target nodes are not in the canonical node space and `analysis/07_figures.py:71-74` has to render them as a separate "synthetic" set, because the manuscript Methods (`manuscript/main.tex:94`) claims target node = "source state $\cup$ {drug class tested}" which implies set-typed semantics that the code does not implement.** Either implement the set semantics or describe the actual string-concatenation behaviour.

29. **ACTION: add a one-line clarification to Methods (`manuscript/main.tex:99-104`) that the concordance algorithm operates edge-by-edge against each guideline node independently and does not traverse the trial DAG, because the current "graph-theoretic" framing oversells the structural use of the DAG; in the implementation the DAG is a visualisation artefact, and `supports()` (`analysis/05_compute_efdpr.py:134`) only inspects single-edge attributes.** This does not weaken the contribution but it should be honest.

30. **ACTION: clarify in Methods that the "guideline-version year" used for ESMO node G7 (post-CDK4/6i PIK3CAmut) is 2022 (`analysis/03_encode_guideline.py:101`) but the guideline source is described as "ESMO 2024 update" (`manuscript/main.tex:88`), because mismatched per-node years allow temporal precedence to silently shift from the analyst, and a reviewer will read 2024 in the prose and 2022 in the code without explanation.** Either justify each per-node year against the published guideline version or fix to a single guideline freeze year.

---

## Verdict

**MAJOR** — the manuscript's quantitative claims rest on a stack that contains at least one fabricated method description (LLM eligibility pipeline that does not exist), one code–paper mismatch on the central state-match rule, undisclosed deviations from the preregistration that change a "rejection" headline to a "marginal" one under the prereg's own test, and supplementary tables promised but not delivered; the analysis is salvageable and the framework is genuinely novel, but the current text is not publishable until the code and prose are reconciled and the disclosed-deviations paragraph is added.
