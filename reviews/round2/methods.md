## 2. New issues introduced in Round 2 (must fix; blocking)

1. **ACTION: reconcile the corpus-size labels across artefacts, because `analysis/07_figures.py:85` still hard-codes the figure title `"HR+/HER2- mBC pivotal-trial DAG (N=14 trials)"` while `main.tex:118`, the abstract, and `tab_cohort.tex` all say 16 trials.** A reviewer who renders the PDF will see the discrepancy on Figure 1; this is a one-line fix but it is the kind of inconsistency that turns into a withdrawal-and-resubmit at JCO CCI. Fix the format-string at `07_figures.py:85` to read from `len(TRIALS)` and rerun.

2. **ACTION: justify the addition of the gBRCAm node (G16) and OlympiAD against the preregistered scope "HR+/HER2- mBC" (`docs/prereg.md:7,16,22,30`), because OlympiAD enrolled gBRCAm metastatic breast cancer regardless of HR status (~50% HR+/HER2-, ~50% HR-/HER2-) and the trial endpoints in the OlympiAD primary publication are reported for the all-comers gBRCAm population, not the HR+/HER2- subset.** Specifically:
    - `02_extract_eligibility.py:179-187` records OlympiAD's biomarker as `"hr_pos": None, "her2_neg": True, "brca_germline": True` — i.e. the trial is admitted on the basis of HER2-negative+gBRCAm, *not* HR+. This is an explicit relaxation of the preregistered cohort restriction "HR+/HER2- mBC" and is nowhere disclosed as a deviation.
    - `03_encode_guideline.py:189` encodes G16's biomarker as `"gBRCAmut/HER2-"` (i.e. drops the HR+ requirement at the guideline node) to permit the OlympiAD edge to match. This is a guideline-encoding choice with no source citation in the supplement.
    - The result is that one of the 11 evidence-supported nodes (G16) is supported only because the prereg's HR+ requirement was dropped at this single guideline node. Under a strict reading of the prereg, G16 should either (i) be excluded from the corpus (with EFDPR recomputed on 15 nodes), or (ii) be admitted with the HR+ subgroup readout of OlympiAD invoked under the *liberal* tolerance and *not* the strict tolerance. Currently G16 is supported under *all three* tolerances (`efdpr.json:59,121,184`), which inflates strict-EFDPR evidence support by 1/16 = 0.0625, the same magnitude as the EFDPR change between strict (0.31) and liberal (0.25).
    - **At minimum** add a "Deviations from preregistration" bullet to `supplement.tex` disclosing the gBRCAm-node scope expansion; **better**, run a pre-specified-scope sensitivity analysis with G16 dropped (15-node EFDPR) and report it in Table 2.

3. **ACTION: tighten the drug-class match for `"CDK4/6i + ..."` strings, because Round 1 fix introduced a new collision risk: the corpus now contains `"CDK4/6i + AI"` (MONALEESA-2/3/7, PALOMA-2), `"CDK4/6i + fulvestrant"` (PALOMA-3, MONARCH-2), `"CDK4/6i + fulvestrant (post-CDK4/6i)"` (postMONARCH), and `"CDK4/6i + endocrine (pre-menopausal)"` (MONALEESA-7), and `05_compute_efdpr.py:123` still does `edge_class.startswith("CDK4/6i") and rc.startswith("CDK4/6i")`.** The blast radius is currently absorbed by state-match (e.g. postMONARCH does not collide with G1 because its source state is `post-endo+post-CDK46i`), but the drug-match rule itself is now structurally permissive and one re-encoding of a trial's prior state could silently change a downstream EFDPR by a full $1/16$. Replace the prefix-rule block (`05_compute_efdpr.py:122-132`) with an explicit equivalence table — for example, a dict mapping every observed edge-class string to a set of guideline-class strings it is allowed to satisfy — and publish that table as Supplementary Table S4b. The five-line prefix shortcut should not survive into a release artefact.

4. **ACTION: scope the `HER2-ADC` and `SERD oral` prefix-match cases the same way, because `05_compute_efdpr.py:125` accepts any edge class starting with `"HER2-ADC"` as concordant with any guideline class starting with `"HER2-ADC"`, and the corpus already differentiates T-DXd from generic HER2-ADC by the parenthetical `"HER2-ADC (T-DXd)"`.** A future trial with `"HER2-ADC (T-DM1)"` or `"HER2-ADC (datopotamab)"` will silently pass through the same prefix rule. The same concern applies to `"SERD oral"` (currently single-trial). Use the explicit equivalence table from R2-#3.

5. **ACTION: add the corpus-selection-bias disclosure to Methods §2.1 (not just buried in the Discussion or supplement), because the trial corpus is drawn from "ESMO and ASCO mBC guideline reference lists" (`main.tex:118`) and the analysis outcome is guideline–trial concordance; this is structurally circular and is the single biggest external-validity threat to the EFDPR estimate.** Currently the Methods paragraph is silent and the supplement-deviation list (`supplement.tex:104-108`) acknowledges the limitation but defers a search-based corpus to the "production-scale extension". This needs to be in the main-text Methods so a JCO CCI reviewer sees the conditional nature of the EFDPR estimate without reading the supplement. The corresponding sensitivity is to also report `guideline-conditional EFDPR` as the headline label for this pilot.

## 3. Statistical-inference issues that survived Round 1 (re-asked)

6. **ACTION: add a state-equality sensitivity analysis alongside strict/ESCAT/liberal, because the superset rule is now correctly described in Methods (`main.tex:100`) but the manuscript does not report what EFDPR would be under the alternative state-equality rule (which is the rule a reader of the prereg, written before Round 1, would expect).** This is the Round 1 ask #3, now narrowly re-scoped: compute EFDPR under state-equality and add a one-line row to Table 2 or a sentence to §3.4 (Sensitivity analyses). Without this number, the reader cannot calibrate how much of the headline 0.31 is driven by the superset relaxation.

7. **ACTION: address the MONALEESA-3 line-of-therapy ambiguity in Methods, because `02_extract_eligibility.py:101-105` now encodes `post_endo: True` (Round 1 fix) but the inline comment honestly notes the trial enrolled both 1L and post-endo populations and chose the modal stratum.** This means MONALEESA-3 supports G3 in `efdpr.json:30` but is silently excluded from G1, when in fact the trial's registrational outcome included both. Either (a) add MONALEESA-3 as an edge from `first-line` (matching G1) in addition to its current `post-endo` edge (matching G3), with a justification, or (b) disclose in Methods that "modal-stratum" encoding is the rule for trials with mixed-line enrolment and that this encoding choice biases EFDPR in a specified direction.

8. **ACTION: either delete the `excluded` field from `02_extract_eligibility.py` or wire it into `05_compute_efdpr.py`, because as Round 1 already noted the field is still populated (e.g. PALOMA-3: `excluded: {"post_cdk46i": True}`, line 35; MONARCH-2: `excluded: {"post_cdk46i": True, "post_chemo_metastatic_any": True}`, line 45) and still never read.** A trial that *excludes* a prior state is currently equivalent in the concordance algorithm to a trial that is *silent* on that state, and the manuscript does not say so. This was Round 1 ask #6; it remains a code-paper gap.

9. **ACTION: replace `int(0.025*n)-1` / `int(0.975*n)` indexing (`05_compute_efdpr.py:176-177`) with `numpy.quantile(estimates, [0.025, 0.975], method='linear')` and document the chosen method in Methods, because the present indexing is closer to textbook than the Round 1 version but is still not the standard interpolated-percentile estimator and still differs by one estimate at the boundaries when many bootstrap replicates collapse to the same $k/n$ value (which is exactly the present regime).** This is a small, demonstrable difference at $n=16$; either fix it or justify why a discrete-attainable-values indexing is preferable here.

## 4. ODI methodology

10. **ACTION: define "registrational biomarker-subgroup readout" in Methods §2.5 against the explicit field `subgroup_readouts` populated in `02_extract_eligibility.py`, because the term is currently used in three places (`main.tex:55`, `main.tex:100`, `main.tex:135`) without a tie to a data field a reader can audit.** Supplementary Table S4 should explicitly list which `subgroup_readouts` token was credited for each trial (EMERALD's `"ESR1mut"` rescues G5 under liberal; CAPItello-291's `"AKTpath"` does not rescue G6 because state-match still fails). This was Round 1 ask #14.

11. **ACTION: change the Methods sentence "treating each definition as a set of operational tokens parsed from the eligibility text" (`main.tex:107`) to "treating each definition as a set of operational tokens hand-coded from the canonical primary publication of each trial against the fixed token vocabulary in Supplementary Table S4", because `06_compute_odi.py:29-57` is a static dict, not a parser, and `02_extract_eligibility.py` is the same; nothing in the repository parses anything.** This is the same overclaim Round 1 #16 flagged in a different sentence; Round 2 fixed the trial-eligibility version but not the ODI-tokens version.

12. **ACTION: re-examine the HER2-low ODI value 0.200 against what DESTINY-Breast06 actually reports, because `06_compute_odi.py:30-33` encodes DESTINY-Breast06's HER2-low definition as `{"IHC1+", "IHC2+_ISH_negative", "central_read", "ISH_threshold_2", "HER2_ultralow_eligible"}` and DESTINY-Breast04 as the same set minus `"HER2_ultralow_eligible"`.** Two problems:
    - The Modi et al. 2024 NEJM primary publication for DESTINY-Breast06 reports the HER2-low population (IHC1+ or IHC2+/ISH-) and the HER2-ultralow population (IHC0 with membrane staining) as separate enrolment strata with separate efficacy readouts; conflating both under a single `HER2_ultralow_eligible` token does not represent the DESTINY-Breast06 design.
    - The manuscript Results (`main.tex:152`) describes the inter-trial difference as "DESTINY-Breast06's inclusion of HER2-ultralow", which suggests an HER2-low+ultralow union, but the ODI computation is on a single token. Either (a) split the DESTINY-Breast06 definition into two trial-rows ("DESTINY-Breast06 HER2-low" and "DESTINY-Breast06 HER2-ultralow") and recompute ODI, or (b) add an explicit "HER2-low+ultralow" composite biomarker variable to Table 3 with its own ODI, or (c) explicitly state in the Results paragraph that the 0.200 figure is for the HER2-low *enrollment criterion* only, not the readout strata.

13. **ACTION: flag the singleton-Jaccard mechanical-inflation issue for the prior-CDK4/6i ODI as a Methods limitation, because the per-trial token sets in `06_compute_odi.py:48-56` are still mostly singletons** (`{"prior_CDK4_6i_required"}` for EMERALD/postMONARCH, `{"prior_CDK4_6i_allowed"}` for CAPItello-291, `{"prior_endo_progression"}` for PALOMA-3/MONARCH-2, `{"endo_resistance_window_12mo"}` for INAVO120, two-element for DESTINY-Breast06), and Jaccard distance is 1 whenever singletons differ. The headline ODI = 0.905 is therefore a categorical-disagreement-count proxy, not a graded operational-discordance measure. This is Round 1 ask #22; the supplement deviation list still does not flag it. At minimum, change `discussion.tex:5` or the limitations paragraph to acknowledge that "the prior-CDK4/6i ODI is dominated by binary disagreement on a single canonical token per trial and overstates the *graded* operational discordance".

## Verdict

**MINOR** — the four Round 1 blockers are now resolved (LLM fabrication, state-superset description, NCT identifiers, pre-registered exact-binomial test) and the manuscript correctly reports a non-rejection ($P=0.37$). The OlympiAD/G16 expansion is the only new methodological move that needs a substantive fix (R2-#2: disclose the gBRCAm scope deviation and ideally report the 15-node sensitivity); everything else on this list is documentation tightening, an equivalence table for drug-class match, and rendering S2/S3/S4 as actual tables. The science is now defensible; the artefact bundle needs one more pass before submission.
